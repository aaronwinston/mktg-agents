# Technical accuracy rubric

## Score definitions

### Score 5 (Excellent)
Every technical claim is verified. Terms are used precisely. Code is syntactically correct and idiomatic. Tradeoffs and limitations are acknowledged where appropriate.

### Score 3 (Acceptable)
Most technical claims are correct. One or two terms are used imprecisely. Code is functional but not idiomatic. Major tradeoffs are acknowledged.

### Score 1 (Poor)
Technical claims are unverified or incorrect. Terms are confused. Code does not work. Tradeoffs are ignored.

## Reviewer questions

- Are all product capability claims verified against documentation?
- Are technical terms (eval, trace, span, observability) used precisely?
- Is code syntactically correct and idiomatic for the language?
- Are benchmark or performance claims sourced?
- Are known limitations acknowledged?

## Common failure modes

- Confusing observability with evaluation
- Confusing monitoring with debugging
- Unsourced benchmark claims
- Incorrect or non-idiomatic code
- Ignoring known limitations or tradeoffs
