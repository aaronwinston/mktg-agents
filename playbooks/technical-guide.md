# Technical guide playbook

## Use this playbook for
- How-to guides
- Integration guides
- Step-by-step tutorials for developers
- Workflow documentation

## Input

Use `briefs/technical-guide-brief-template.md`.

## Skill chain

```
technical guide brief
→ editorial-director
→ ai-researcher
→ dev-copywriter
→ dev-reviewer
→ technical-fact-checker
→ copy-chief
→ final-publish-reviewer
```

## Steps

### 1. Editorial direction
Use `skills/editorial/editorial-director/SKILL.md`.

### 2. Research
Use `skills/foundation/ai-researcher/SKILL.md`.

### 3. Draft
Use `skills/foundation/dev-copywriter/SKILL.md`.
Technical guides require concrete workflow steps and code examples.

### 4. Developer review
Use `skills/foundation/dev-reviewer/SKILL.md`.

### 5. Technical fact-check
Use `skills/specialization/technical-fact-checker/SKILL.md`.
All code and technical claims must be verified.

### 6. Copy edit
Use `skills/editorial/copy-chief/SKILL.md`.

### 7. Final publish review
Use `skills/quality/final-publish-reviewer/SKILL.md`.

## Required rubrics
- `rubrics/technical-accuracy.md`
- `rubrics/developer-fluency.md`

## Done means
- All code and technical claims are verified
- Workflow steps are complete and accurate
- Copy is polished and ready to publish
