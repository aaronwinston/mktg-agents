# Product launch playbook

## Use this playbook for
- Product releases
- Feature launches
- Integration announcements
- Partner launches

## Input

Use `briefs/launch-brief-template.md`.

## Skill chain

```
launch brief
→ competitive-intelligence   [proactive scan: competitive surface area + framing]
→ pmm-lead                   [positioning, message hierarchy, ICP fit]
→ editorial-director
→ launch-comms-writer
→ competitive-intelligence   [review: land mines, differentiation gaps]
→ technical-fact-checker
→ claims-risk-reviewer
→ narrative-consistency-reviewer
→ copy-chief
→ final-publish-reviewer
→ social-editor
→ content-ops-manager
```

## Steps

### 1. Competitive scan
Use `skills/specialization/competitive-intelligence/SKILL.md`.
Before any strategy or writing begins, check `context/02_narrative/competitive-pov.md` for relevant research.
Output must include: competitive surface area rating, relevant competitor narratives, land mines to avoid, differentiation opportunities.

### 2. PMM positioning
Use `skills/specialization/pmm-lead/SKILL.md`.
Output must include: positioning served, ICP fit, message hierarchy, competitive framing, recommended changes to brief.

### 3. Editorial direction
Use `skills/editorial/editorial-director/SKILL.md`.
Define the positioning angle, why this matters now, and the proof points needed.

### 4. Launch copy
Use `skills/specialization/launch-comms-writer/SKILL.md`.
Output must include: announcement copy, blog intro, email blurb, social copy, executive quote, short and long versions.

### 5. Competitive review
Use `skills/specialization/competitive-intelligence/SKILL.md`.
Review the draft for land mines, neutralizable claims, and unused differentiation angles.
Output must include: competitive sharpness rating, flagged claims with rewrites, verdict.

### 6. Technical fact-check
Use `skills/specialization/technical-fact-checker/SKILL.md`.
Every product capability claim must be verified against product documentation.

### 7. Claims review
Use `skills/quality/claims-risk-reviewer/SKILL.md`.
Every customer, benchmark, performance, and competitive claim must be reviewed.

### 8. Narrative consistency
Use `skills/quality/narrative-consistency-reviewer/SKILL.md`.
Confirm the launch copy aligns with company narrative and category positioning.

### 9. Copy edit
Use `skills/editorial/copy-chief/SKILL.md`.

### 10. Final publish review
Use `skills/quality/final-publish-reviewer/SKILL.md`.
A launch requires explicit sign-off before any asset goes live.

### 11. Distribution
Use `skills/specialization/social-editor/SKILL.md`.

### 12. Operations
Use `skills/editorial/content-ops-manager/SKILL.md`.

## Required rubrics
- `rubrics/launch-readiness.md`
- `rubrics/technical-accuracy.md`
- `rubrics/brand-fit.md`
- `rubrics/distribution-readiness.md`

## Done means
- All assets reviewed by product, technical, and claims reviewers
- Executive approval confirmed
- Launch timing confirmed
- Distribution plan ready
- Publishing checklist complete
