# Case study playbook

## Use this playbook for
- Customer case studies
- Technical success stories
- Use case narratives

## Input

Use `briefs/case-study-brief-template.md`.

## Skill chain

```
customer idea
→ customer-story-producer
→ editorial-director
→ claims-risk-reviewer
→ copy-chief
→ final-publish-reviewer
→ social-editor
```

## Steps

### 1. Story assessment
Use `skills/specialization/customer-story-producer/SKILL.md`.
Do not proceed without confirming customer approval status.

### 2. Editorial direction
Use `skills/editorial/editorial-director/SKILL.md`.

### 3. Claims review
Use `skills/quality/claims-risk-reviewer/SKILL.md`.
Every customer result, metric, and quote must be approved by the customer.

### 4. Copy edit
Use `skills/editorial/copy-chief/SKILL.md`.

### 5. Final publish review
Use `skills/quality/final-publish-reviewer/SKILL.md`.
Confirm customer approval before this step.

### 6. Distribution
Use `skills/specialization/social-editor/SKILL.md`.

## Required rubrics
- `rubrics/editorial-quality.md`
- `rubrics/brand-fit.md`

## Done means
- Customer has approved all quotes, results, and references
- Claims are verified
- Legal review complete if required
- Distribution plan ready
