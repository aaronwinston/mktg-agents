# 60-Day Content Execution: Quick Reference

## 📋 DAILY CHECKLIST TEMPLATE

Use this for each article day:

### Pre-Writing (Day Before)
- [ ] Topic & keywords identified
- [ ] Outline drafted (5 key sections)
- [ ] 3-5 sources/examples identified
- [ ] Style guide + tone reference ready

### Writing Day (Morning)
- [ ] Deep outline with specific examples
- [ ] First 2 sections written
- [ ] Graphics/code examples planned

### Writing Day (Afternoon)
- [ ] Remaining sections written
- [ ] All code examples added
- [ ] Graphics created/sourced

### Publishing Day
- [ ] Proofread (typos, clarity, flow)
- [ ] Format for WordPress
- [ ] Add SEO metadata
- [ ] Extract Twitter thread
- [ ] Create 3-5 social clips
- [ ] Schedule distribution

## 📝 ARTICLE STRUCTURE TEMPLATE

```
# [HEADLINE]

## Introduction (500 words)
- Hook: Why this matters
- Problem statement
- Scope & what reader will learn

## Section 1: Context (600-800 words)
- Background
- Industry trends
- Why now?

## Section 2: Deep Dive (800-1000 words)
- Core concept explained
- Examples or case study
- Key patterns

## Section 3: Practical Application (800-1000 words)
- How to implement
- Code examples
- Best practices

## Section 4: Advanced Topics (600-800 words)
- Edge cases
- Pitfalls to avoid
- When NOT to use this

## Conclusion (400-500 words)
- Summary of key points
- Call to action (read next article, try tool, etc.)
- Future implications

## Metadata (for WordPress)
- SEO title
- Meta description
- Keywords (7 keywords, including agent/harness terms)
- Featured image
- Category & tags
```

## 🐦 TWITTER THREAD EXTRACTION

**From each article, create 1 thread:**

1. Identify 12-15 key insights
2. Order them logically (problem → solution → action)
3. Expand to full thoughts (280 chars max per tweet)
4. Add 1-2 graphics for key tweets (#5, #10)
5. Include CTA at end (link to article, code repo, etc.)

**Thread Formula:**
```
Tweet 1: Hook (problem statement, surprise, question)
Tweets 2-3: Why it matters (cost, risk, opportunity)
Tweets 4-7: The core insight (broken into steps)
Tweets 8-11: Practical application (code, example, pattern)
Tweet 12: CTA (read full article, try it, ask questions)
```

## 📊 SOCIAL CLIP TEMPLATE (Per Article)

Create 3-5 clips:
1. **Quote clip:** Pull strongest quote + author photo
2. **Data clip:** If applicable, 1-2 stat/metric from article
3. **Definition clip:** Key concept explained in 1 sentence
4. **How-to clip:** 1 actionable step from article
5. **Vision clip:** Future-looking insight

**Design:**
- 1200x630px for LinkedIn/Twitter
- Clean typography (2-3 colors max)
- Your name/handle + date
- Source link small in corner

## ✅ PRE-PUBLICATION CHECKLIST

Before publishing any article:

- [ ] **Content:**
  - [ ] Spell-checked (Grammarly, etc.)
  - [ ] Fact-checked (claims verified)
  - [ ] Code examples tested (ran locally)
  - [ ] Links verified (no 404s)

- [ ] **SEO:**
  - [ ] Keyword in title
  - [ ] Keyword in first 100 words
  - [ ] Meta description written
  - [ ] Internal links added (to other articles)
  - [ ] External links (3-5 credible sources)

- [ ] **Format:**
  - [ ] Heading hierarchy correct (H1 > H2 > H3)
  - [ ] Graphics embedded correctly
  - [ ] Code blocks formatted
  - [ ] Lists formatted consistently

- [ ] **Distribution:**
  - [ ] WordPress import file created
  - [ ] Twitter thread extracted
  - [ ] Social clips generated (3-5)
  - [ ] LinkedIn summary written
  - [ ] Dev.to/Hacker News version prepared

- Top 3-5 topics by performance
- Keyword ranking (Google Search Console)
- Inbound leads/inquiries
- Social following growth

**60-Day (End of Month 2):**
- Total reach/impressions
- Conversion metrics (if tracked)
- Authority metrics (backlinks, mentions)
- Next phase recommendations

## 💡 CONTENT REPURPOSING MULTIPLIER

**1 Article = 6 Distribution Pieces:**
1. Company blog post
2. Twitter thread
3. LinkedIn summary post
4. Dev.to discussion
5. Newsletter excerpt
6. Slack community post

**This 1:6 ratio means 16 articles = 96 total distribution pieces**

---

## 🎯 KEYWORD DENSITY TARGETS

Per article: Aim for 0.8-1.2% keyword density for primary keyword

**Example:** For 2K-word article (400 words per keyword target)
- Primary keyword: 16-24 mentions (~1%)
- Secondary keywords: 8-12 mentions each (~0.5%)

Spread naturally throughout (intro, section headers, conclusion, meta).
